# Help Menu

Reached from the Main Menu with `help`. This is PRISM's built-in
documentation — start here if you're new.

## Sections

- `readme` — the same onboarding tips shown when PRISM starts up (how
  commands work, how to chain them, how to save shortcuts). Use
  `readme set` under `settings` if you want to stop it appearing on startup.
- `general` — a short plain-English summary of what PRISM is for.
- `ra` — **Research Assistant documentation.** This is the section written
  for you. It covers:
  - `start` — what folders/files need to exist for PRISM to run
  - `navigation` — how to move around using commands, not just menus
  - `navigation advanced` — command chaining and saved shortcuts (macros)
  - `terminals` — the four different prompts you'll see (`prism>`,
    `assistant>`, `twilio>`, and `ENTER to Continue>`) and what each does
  - `task management` — the basics of scheduling/running tasks
  - `participant management` — the basics of adding/editing participants
- `dev` — developer-facing documentation (API endpoints, config file
  schemas, backend architecture). Only worth reading if you're doing
  technical/administrative work on PRISM itself, not day-to-day study
  operation.

## Recommended reading order

If you're brand new to PRISM: `help ra start`, then `help ra navigation`,
then `help ra terminals`. Everything else you can look up as you need it —
or just ask the `assistant` (see `assistant/README.md`).
